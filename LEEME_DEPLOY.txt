  🚀 PASOS RÁPIDOS PARA DEPLOY
═══════════════════════════════════════════════════════════════════════════

  1️⃣  Crear MySQL en la nube
      └─ PlanetScale (gratis): https://planetscale.com/
      └─ O Railway ($5/mes): https://railway.app/

  2️⃣  Subir código a GitHub
      └─ git add .
      └─ git commit -m "Deploy a Vercel"
      └─ git push origin main

  3️⃣  Importar en Vercel
      └─ https://vercel.com/new
      └─ Import Git Repository
      └─ Seleccionar tu repo: EDM04

  4️⃣  Configurar Variables de Entorno en Vercel
      └─ DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD
      └─ SECRET_KEY (generar con: python -c "import secrets; print(secrets.token_hex(32))")
      └─ FLASK_ENV=production

  5️⃣  Deploy
      └─ Click en "Deploy"
      └─ Esperar 2-3 minutos
      └─ ✅ Tu app en https://tu-proyecto.vercel.app

  6️⃣  Inicializar Base de Datos
      └─ Ejecutar database/schema.sql en tu MySQL en la nube
      └─ Crear usuario administrador con SQL

  7️⃣  Verificar
      └─ Abrir https://tu-proyecto.vercel.app
      └─ Probar login
      └─ ✨ ¡Listo!

═══════════════════════════════════════════════════════════════════════════
  ⚠️  IMPORTANTE ANTES DE DEPLOY
═══════════════════════════════════════════════════════════════════════════

  ❌ NO SUBIR archivo .env a Git
     └─ Ya está en .gitignore ✅

  ❌ NO usar SQLite en Vercel
     └─ Usar MySQL en la nube (PlanetScale/Railway)

  ❌ NO contar con persistencia de archivos
     └─ Uploads (PDFs) necesitan Vercel Blob / Cloudinary / S3
     └─ Ver NOTAS_VERCEL.md para soluciones

  ✅ Generar SECRET_KEY segura (32+ caracteres)
     └─ python -c "import secrets; print(secrets.token_hex(32))"

  ✅ Verificar que todas las variables de entorno estén en Vercel
     └─ No en el código, solo en Vercel Dashboard


═══════════════════════════════════════════════════════════════════════════
  🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

  ❓ Error: "Application Error"
     └─ Ver logs: vercel logs --follow
     └─ O en Vercel Dashboard → Deployments → View Function Logs

  ❓ Error: "Can't connect to database"
     └─ Verificar variables de entorno en Vercel Dashboard
     └─ Probar conexión desde local con mismas credenciales

  ❓ Página en blanco
     └─ Ver consola del navegador (F12)
     └─ Verificar que archivos estáticos carguen

  ❓ Cold start lento (primera request)
     └─ Normal en serverless (3-5 segundos)
     └─ Se acelera después del primer request
